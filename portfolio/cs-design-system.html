01 — Problem
Four teams, four inconsistent products

Each product team had built their own UI from scratch. Buttons looked different across apps. The same modal pattern had 6 implementations. Onboarding a new designer meant weeks of archaeology through scattered Figma files.

The engineering cost was even steeper — duplicate components in every codebase meant bug fixes had to be applied 4–6 times.

02 — Foundation
Tokens first, everything else second

The system starts with design tokens — color, spacing, radius, typography — defined once and referenced everywhere. Any global update cascades automatically.

03 — Process
Audit → Consolidate → Document → Adopt
1
Component audit across all products
Catalogued every UI pattern across 4 products. Found 340 unique "components" that consolidated down to 62 base patterns.
2
Figma Variables + Auto Layout rebuild
Rebuilt every component using Figma Variables for tokens and Auto Layout for responsiveness. Each component has documented variants and states.
3
Engineering handoff & code pairing
Worked with a frontend engineer to ensure 1:1 parity between Figma components and the React component library.
4
Documentation & contribution guidelines
Wrote usage guidelines, do/don't examples, and a contribution process so the system grows sustainably without me as a bottleneck.
