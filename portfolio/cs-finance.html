01 — Problem
Users couldn't find what they needed

The existing dashboard buried critical actions under 4–5 taps. Users routinely called support to complete tasks they should be able to do in seconds. Session recordings showed 70% of users abandoned mid-flow on the transfer screen.

The core issue wasn't missing features — it was information architecture. Everything existed, but nothing was where users expected it to be.

📊
Before — cluttered dashboard with 12+ primary actions visible at once
02 — Process
Research → Define → Design → Test
1
User interviews & session recordings
20 interviews with active users. Identified 3 core jobs-to-be-done: check balance, send money, view transactions.
2
Card sorting & IA restructure
Remote card sorting with 40 participants. Rebuilt navigation hierarchy around user mental models, not product features.
3
Rapid prototyping & usability testing
3 rounds of prototype testing with 5 users each. Iterated on task flows until completion rate exceeded 90%.
4
Handoff & implementation support
Worked alongside engineers during a 6-week sprint, resolving edge cases and maintaining design integrity.
03 — Solution
One screen, three actions, zero confusion

The redesigned dashboard surfaces only what matters: current balance, quick-send shortcuts, and a smart transaction feed. Secondary features move to a persistent bottom nav — always one tap away, never in the way.
